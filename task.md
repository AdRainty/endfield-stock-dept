# 终末地调度券交易模拟系统 - 开发任务列表

## 需求分析

### 业务背景
基于《明日方舟：终末地》游戏设定，创建一个调度券交易模拟系统。调度券是游戏中的资源，玩家在两个交易所（四号谷底、武陵）进行交易。

### 核心特性
1. **双交易所**：四号谷底、武陵，各自独立行情
2. **档口交易**：类似券商的买卖五档行情
3. **实时价格**：价格持续波动，无集合竞价
4. **权限分离**：普通用户交易/管理员分配资源
5. **新手福利**：注册赠送每个交易所 10W 调度券

## 撮合引擎重构详情（2026-03-05）

### 新撮合机制
- **买单队列**：`matching:bid:{exchangeId}_{instrumentCode}`
  - Score = 价格 × 1000000 + (MAX_TIME - 时间戳) / 1000000
  - 价格越高、时间越早，分数越高，排在前面

- **卖单队列**：`matching:ask:{exchangeId}_{instrumentCode}`
  - Score = 价格 × 1000000 + 时间戳 / 1000000
  - 价格越低、时间越早，分数越低，排在前面

### 交易时间
- 早盘：9:30 - 11:30
- 午盘：13:00 - 15:00
- 提前 2 分钟启动撮合线程（9:28、12:58）

### 撮合逻辑
- 每 100ms 检查一次最佳买卖单
- 买价 >= 卖价时成交
- 成交价格取卖价（A 股规则）
- 撮合数量取两者较小值
- 支持部分成交和完全成交

## 新闻系统详情（2026-03-05）

### 新闻数据库
```sql
CREATE TABLE news (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(500) NOT NULL,
    content TEXT NOT NULL,
    exchange_id BIGINT,
    instrument_code VARCHAR(50),
    news_type INTEGER,
    sentiment_level INTEGER,
    impact_percent DOUBLE PRECISION,
    processed BOOLEAN DEFAULT false,
    publish_time TIMESTAMP,
    source VARCHAR(100),
    reference_ids TEXT
);
```

### 定时任务
| 时间 | 任务 | 说明 |
|------|------|------|
| 8:30 | 生成早报 | 为两个交易所生成早报新闻 |
| 9:15 | 集合竞价 | 根据新闻影响更新品种价格（工作日） |
| 21:00 | 生成晚报 | 为两个交易所生成晚报新闻 |

### 利好利空等级
| 等级 | 值 | 影响幅度 |
|------|-----|----------|
| 重大利空 | -3 | -25% ~ -30% |
| 利空 | -2 | -15% ~ -25% |
| 偏空 | -1 | -5% ~ -15% |
| 中性 | 0 | 0% |
| 偏多 | 1 | 5% ~ 15% |
| 利好 | 2 | 15% ~ 25% |
| 重大利好 | 3 | 25% ~ 30% |

### 集合竞价规则
- 时间：每个交易日 9:15
- 根据未处理新闻计算各品种累计影响幅度
- 更新品种开盘价
- 卖单低于竞价价、买单高于竞价价的订单全部以竞价成交
- 量化逻辑：存在 10% 概率利好变利空、利空变利好

### 禁止撤单规则
- 时间范围：20:00 - 次日 9:30
- 此时间段内撤单返回错误提示
- 保证隔夜订单稳定性
